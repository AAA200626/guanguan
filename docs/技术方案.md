# 技术方案

> 创建日期：2026-07-18 | 状态：已确定

## 技术选型

### 框架：Flutter 3.x + Dart

- **优点**：一套代码 Android + iOS，Material Design 组件丰富，热重载开发效率高，个人开发者友好
- **缺点**：安装包体积较大（基础 ~15MB），初次编译较慢
- **选型理由**：用户可能后续上 iOS，Flutter 避免重写

### 状态管理：Riverpod

- **优点**：类型安全，编译期检查，无 BuildContext 依赖，比 Provider 更现代
- **对比**：Provider（简单但类型弱）/ Bloc（学习成本高）/ Riverpod（平衡）

### 本地存储

| 数据类型 | 方案 | 原因 |
|----------|------|------|
| 存钱记录、打卡数据 | SQLite (sqflite) | 结构化数据，需查询统计 |
| 用户设置、偏好 | SharedPreferences | 简单键值对 |

### 图表：fl_chart

- 开源免费，支持环形图、折线图、柱状图
- 动画效果好，符合马卡龙风格

### 通知：flutter_local_notifications

- Android NotificationChannel 支持
- 定时提醒、每日打卡通知

## 数据存储

### SQLite 表设计

```sql
-- 存钱计划表
CREATE TABLE plans (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  method_id TEXT NOT NULL,        -- 存钱方法标识
  method_name TEXT NOT NULL,      -- 方法名称
  start_date TEXT NOT NULL,       -- 开始日期
  target_amount REAL NOT NULL,    -- 目标金额
  saved_amount REAL DEFAULT 0,    -- 已存金额
  status TEXT DEFAULT 'active',   -- active / completed / paused
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

-- 打卡记录表
CREATE TABLE records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  plan_id INTEGER NOT NULL,
  date TEXT NOT NULL,              -- 打卡日期
  amount REAL NOT NULL,            -- 当天金额
  is_completed INTEGER DEFAULT 0, -- 是否完成
  FOREIGN KEY (plan_id) REFERENCES plans(id)
);

-- 成就徽章表
CREATE TABLE badges (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  description TEXT,
  icon TEXT,
  earned_date TEXT,
  plan_id INTEGER
);
```
